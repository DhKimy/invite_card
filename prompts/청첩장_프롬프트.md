# 모바일 청첩장 코드 생성 프롬프트

AI(예: Claude)에게 그대로 붙여넣어 청첩장 코드를 생성/수정할 때 쓰는 프롬프트 모음입니다.
`{ }` 안을 본인 정보로 채워서 사용하세요.

---

## ① 마스터 프롬프트 (처음부터 새로 만들 때)

```
역할: 너는 프리미엄 모바일 청첩장을 만드는 프론트엔드 개발자다.
목표: GitHub Pages에 바로 올릴 수 있는 "애플 제품 페이지 스타일"의 인터랙티브 모바일 청첩장을 만든다.

[기술 제약]
- 단일 index.html 하나에 HTML/CSS/JS를 모두 인라인으로 작성 (외부 빌드 도구·프레임워크 금지)
- 외부 의존성 없음. 폰트는 시스템 폰트 + Noto Serif KR 정도만
- 모바일 우선. 데스크톱에서도 max-width 480px로 중앙 정렬
- viewport-fit=cover, 100svh 사용. 노치/주소창 대응

[인터랙션 — 반드시 포함]
1. Sticky pin: 한 섹션을 화면에 고정하고 스크롤 진행도로 내부를 연출(scrub)
2. IntersectionObserver 기반 fade-up reveal (같은 섹션은 stagger)
3. 패럴랙스 히어로 (배경이 스크롤보다 느리게 이동)
4. requestAnimationFrame으로 스크롤 핸들러 throttle
5. transform/opacity만 애니메이트 (성능)

[정보 구조 — 이 순서대로 섹션 구성]
1. 커버: {신랑이름} & {신부이름}, {예식일자/요일/시간}, 대표사진
2. 인사말: {초대 문구}
3. 신랑&신부: {신랑 풀네임/혼주}, {신부 풀네임/혼주}, 전화걸기(tel:)
4. 갤러리: 사진 6~9장 (가로 스와이프 + 그리드). 지금은 picsum.photos 플레이스홀더 사용, 교체 주석 표기
5. 캘린더 + D-day: {예식일자} 기준 자동 계산
6. 오시는 길: {예식장명/주소}, 네이버·카카오·티맵 길찾기 버튼
7. 마음 전하실 곳: {신랑측 계좌}, {신부측 계좌}, 복사 버튼 + 토스트, 접기형(details)
8. 마무리 + 공유하기 (navigator.share, 폴백은 링크 복사)

[디자인]
- 무드: {예: OLED 다크 + 골드 / 크림 + 더스티골드 / 블러쉬 로즈}
- 큰 세리프 타이포, 여백 넉넉히, 절제된 색
- 코드에 한국어 주석으로 각 기법 설명

산출물: 완성된 index.html 전체 코드 하나.
```

```

- {예: 다크 테마를 크림+세이지그린으로 바꿔줘}
- {예: 갤러리를 12장으로 늘리고 3열 그리드로}
- {예: RSVP(참석 의사) 폼 섹션을 캘린더 다음에 추가. 이름/참석여부/인원}

```

## ④ 기능별 추가 스니펫 요청 프롬프트

- **카카오 지도 연동**: `"오시는 길 섹션의 지도 플레이스홀더를 Kakao Maps JS SDK로 교체하는 코드 줘. 좌표는 {위도},{경도}, 마커 표시. API 키 자리는 주석으로."`
- **RSVP 구글폼/시트 연동**: `"RSVP 폼 제출을 Google Apps Script 웹앱으로 보내는 코드 줘. 필드는 이름/참석여부/동행인원/식사여부."`
- **카카오톡 공유**: `"Kakao SDK(shareDefault)로 청첩장 링크를 카카오톡 공유하는 버튼 코드 줘. 썸네일/제목/설명 포함."`
- **BGM**: `"우측 하단에 음소거 토글이 있는 자동재생 배경음악 추가. 모바일 자동재생 정책 고려해 첫 탭에서 재생 시작."`
- **접근성**: `"prefers-reduced-motion 사용자를 위해 모든 스크롤 애니메이션을 끄는 미디어쿼리/JS 가드를 추가해줘."`

## ⑤ GitHub Pages 배포 안내 (사람이 직접)

```
1. GitHub에서 새 저장소 생성 (예: wedding-invitation), Public
2. index.html 을 저장소 루트에 업로드 (이 프로젝트의 샘플 하나를 루트로 복사)
3. Settings → Pages → Source: "Deploy from a branch" → main / root 선택 → Save
4. 1~2분 후 https://{아이디}.github.io/{저장소명}/ 으로 접속
5. 사진은 저장소에 images/ 폴더로 올리고 코드의 picsum.photos URL 을 교체
```
> 루트 index.html 하나만 있으면 끝. 별도 빌드 불필요.
```
