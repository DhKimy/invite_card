## 2. 모바일 청첩장 표준 섹션 (정보 구조)

거의 모든 우수 사례가 아래 순서를 따릅니다. 샘플 3종 모두 이 골격을 공유합니다.

1. **커버 / 히어로** — 신랑·신부 이름, 예식 날짜, 대표 사진
2. **인사말(초대글)** — 짧은 초대 문구
3. **신랑 & 신부 소개** — 이름, 혼주(부모), 전화 연결
4. **갤러리** — 웨딩/스냅 사진 (가로 스와이프 또는 그리드)
5. **예식 일시 / 캘린더 / D-day(카운트다운)**
6. **오시는 길** — 주소 + 지도 + 길찾기 앱 버튼(네이버·카카오·티맵)
7. **마음 전하실 곳** — 계좌번호 + 복사 버튼 (접기형)
8. **방명록 / RSVP(참석의사)** — 선택
9. **마무리 / 공유하기** — 카카오톡·링크 공유

## 3. 애플 감성을 만드는 인터랙션 기법 (실전)

### A. Sticky Pin (고정 후 연출) — 가장 "애플다운" 기법
- 섹션 높이를 `300vh`처럼 길게 주고, 내부를 `position:sticky; top:0`으로 한 화면에 고정.
- 사용자가 스크롤하는 동안 화면은 멈춰 있고, **내부 요소만** 변한다 → 영화처럼 느껴짐.
- 샘플 01의 "스토리 핀" 섹션 참고.

### B. Scroll Scrub (스크롤 진행도 구동)
- 요소가 화면을 지나는 정도를 `0~1` 진행도로 계산해 CSS 변수(`--p`)로 주입.
- 그 변수로 `transform: scale()`, `filter: brightness()`, `opacity`를 연동.
```js
function progress(el){
  const r = el.getBoundingClientRect();
  return Math.min(1, Math.max(0, -r.top / (r.height - innerHeight)));
}
```

### C. Reveal on Scroll (등장 애니메이션)
- `IntersectionObserver`로 화면에 들어온 요소에 `fade-up` 적용. 같은 섹션은 `transition-delay`로 **계단식(stagger)** 등장.
- 모바일 성능을 위해 `transform`/`opacity`만 애니메이트(레이아웃 리플로우 회피).

### D. Parallax (속도차)
- 배경 이미지를 스크롤 속도보다 느리게 이동(`translateY(scrollY * 0.3)`)시켜 깊이감 부여.

### E. 마이크로 인터랙션
- 봉투 열기, 글자 한 자씩, 계좌 복사 토스트, 도트 네비게이션, 카운트다운 — 작은 반응들이 "공들인 느낌"을 만든다.

### 성능/접근성 체크리스트
- [ ] `requestAnimationFrame`으로 스크롤 핸들러 throttle
- [ ] `transform`/`opacity` 위주 (top/left/width 애니메이션 금지)
- [ ] `prefers-reduced-motion` 존중(멀미 방지) — 추후 추가 권장
- [ ] 이미지 `loading="lazy"`, 적정 해상도
- [ ] viewport `100svh` 사용(모바일 주소창 높이 변화 대응)
- [ ] `viewport-fit=cover`로 노치 대응
