# 모바일 청첩장 — The Editorial

결혼을 한 권의 매거진처럼 다루는 **에디토리얼 무드** 모바일 청첩장입니다.
의존성 없는 **단일 `index.html`** 이라 GitHub Pages에 그대로 올리면 동작합니다.
(인스타 `@thenewchapter.official` 의 무드에서 착안 · 브랜드 비제휴)

## 폴더 구조
```
chungchup_1/
├── index.html                  ← 청첩장 본체 (이 파일 하나가 전부)
├── README.md                   ← 지금 보는 파일
├── references/우수사례_분석.md   ← (참고) 우수 사례·인터랙션 기법 정리
└── prompts/청첩장_프롬프트.md     ← (참고) AI로 수정/생성하는 프롬프트 모음
```

## 인터랙티브 요소
- **커버 깊이 패럴랙스** — 데스크톱은 마우스, **모바일은 자이로(폰 기울임)** 로 표지가 입체적으로 반응
- **가로 핀 갤러리** — 스크롤하면 사진이 옆으로 흐르는 매거진 스프레드
- **사진 현상** — 사진을 호버(PC)/탭(모바일)하면 흑백 듀오톤이 컬러로 살아남
- **속도 반응 리본** — 스크롤 속도에 따라 흐르는 텍스트가 가속·기욺
- **커스텀 커서 + 마그네틱**(PC) / **탭 누름 피드백**(모바일)
- 챕터 워터마크 · 읽기 진행 레일 · 캘린더/카운트다운 · 계좌 복사 · 공유하기

> 모바일 자이로는 **보안 컨텍스트(https)** 가 필요합니다(iOS). 폰에서 제대로 보려면 GitHub Pages 배포(https)로 확인하세요. 모션에 민감한 사용자(`prefers-reduced-motion`)는 장식 모션이 자동 비활성됩니다.

## 로컬 미리보기
이 폴더에서:
```
python3 -m http.server 8000
```
→ 브라우저에서 `http://localhost:8000/` 접속 (모바일 뷰로 보면 가장 잘 보임).

## GitHub Pages 배포
1. GitHub에 새 저장소(Public) 생성 → `index.html` 을 루트에 업로드
2. Settings → Pages → Source: *Deploy from a branch* → `main` / `root` → Save
3. 1~2분 뒤 `https://{아이디}.github.io/{저장소명}/` 으로 접속 (https라 모바일 자이로도 동작)

## 내 청첩장으로 채우기 (다음 단계)
`index.html` 안에서 아래를 본인 정보로 교체합니다.
- 이름 · 혼주 · 날짜 · 시간 · 장소 · 주소 · 전화 · 계좌 · 초대 문구
- 사진: `picsum.photos/...` URL → 실제 사진 경로(`images/` 폴더 권장)로 교체
- 코드 상단 주석에 각 섹션 위치가 표시되어 있고, `prompts/청첩장_프롬프트.md` 의 ③/④ 프롬프트로 AI 수정도 가능

> 현재 `index.html`은 김동현 · 성우민 실제 정보와 사진이 반영된 단일 배포 파일입니다.
